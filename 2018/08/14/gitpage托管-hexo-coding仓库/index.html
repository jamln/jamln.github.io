gitpage托管+hexo+coding仓库

发表于 2018-08-14 更新于 2024-06-03 分类于 笔记

主要想实现的是博客源文件保存在coding 私有仓库，方便在多台设备进行编写博客，hexo d 生成的静态博客文件保存在github公开仓库（主要是因为github 私有仓库收费）

Git Page

网上教程比较多，主要就是在你的github上新建一个以你用户名（此处假定为username）开头加上 .github.io 为名称的仓库，如 username.github.io 。

之后这个 username.github.io 也就自动变成你 gitpage 的访问地址啦。

后面生成的 静态页面文件也就是推送到这个仓库上。

*由于国内访问gitpage的速度比较慢，同时可采用国内的代码仓库托管静态页面如 coding、gitee，操作方式类似*

部署

首先生成本地静态页面文件

首先 ，需要把你的公钥上传到 github 中，这样你才有把代码推送到 gitpage 仓库的权限。

其次 ，需要配置好 博客目录下的 _config.yml 文件才能进行部署，主要修改的是文件最底部的 deploy 字段部分

执行部署，之后便可在 xxx.github.io 看到自己的博客啦

部署前需要先安装插件 $ npm install hexo-deployer-git --save

这个命令会把 $ hexo g 生成的静态页面文件 推送到 github 事先配置的 xxx.github.io 这个项目

Coding

建立仓库保存博客源文件，之后可基于这个仓库和hook进行自动部署，如push源文件代码到这个仓库，触发部署到 gitpage 或 coding page 的动作

coding 也有类似 gitpage 的静态页面托管功能，通过 username.coding.me 域名访问

Coding Page

主要因为coding改版网上很多教程都是基于旧版的，所以稍微提一下coding page 的静态页面托管配置

  1. 新建一个项目，以 username.coding.me 这种形式命名项目的话，上传代码后能直接以 username.coding.me 访问页面了。
  2. 进入项目–>点击展开左侧代码菜单–>最底下有个 “page 服务菜单项”
  3. 进入page 服务 开启服务即可
